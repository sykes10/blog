<objective>
Deliver category and tag browsing for the Patterns track (PATT-04): statically generated per-category routes via `generateStaticParams`, and a client-side tag chip filter on the index. This satisfies the phase-goal capability "browse Pattern posts by category and by tag."

Purpose: Make the growing Patterns library navigable by its two organizing dimensions (the 4-value `category` enum and free-form `tags`) while preserving the site's fully-static, SEO-indexable architecture — category dimensions get their own crawlable URLs; tags get instant no-navigation filtering (Claude's discretion per CONTEXT.md, RESEARCH.md Pattern 2 + 3).

Output: `getPatternsByCategory()`, `getPatternsByTag()`, `getAllTags()` helpers in `lib/content.ts`; a static `/patterns/category/[category]` route with enum validation + `notFound()`; a `"use client"` `<TagFilter>` chip island wired into the `/patterns` index; and Playwright specs for both.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| category route param → data lookup | The `[category]` URL segment is attacker-controllable; it must be validated against the fixed enum before use |
| author frontmatter tags → client filter | Tags originate from author-controlled MDX frontmatter (not user input); the client filter operates over this trusted set |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Tampering | app/patterns/category/[category]/page.tsx route param | medium | mitigate | Validate the `category` param against the fixed CATEGORIES enum (["components","behaviours","engineering","ux"]) and call notFound() on mismatch before any lookup (RESEARCH.md Pitfall 3, Security V5). getPatternsByCategory filters an in-memory array — no filesystem/DB path is built from the param — so traversal risk is inherently low, but explicit enum validation is standard and cheap. Enforced by acceptance grep on notFound. |
| T-02-04 | Injection | TagFilter chip rendering | low | accept | Tag values come from author-controlled frontmatter (velite-validated strings), not user input; React escapes text content by default; no dangerouslySetInnerHTML is introduced. Same trust boundary as the existing index page. |
| T-02-SC | Tampering | npm installs | low | accept | Zero new npm packages this phase (RESEARCH.md Package Legitimacy Audit: not applicable). No install tasks; no [ASSUMED]/[SUS] packages to gate. |
</threat_model>
